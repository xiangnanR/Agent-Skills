# 风格定义 — 企业科技信息图 (Corporate Tech Infographic)

## 风格概述

该风格属于 **"企业科技信息图"（Corporate Tech Infographic）** 风格，广泛应用于 SaaS 营销、技术文档、投资人演示文稿和 AI/ML 产品推广中（约 2020-2024 年流行）。其核心目标是：**将复杂的技术概念通过视觉隐喻转化为直观、易懂、专业的图形表达**。

**语言规范**：图片中出现的所有文字（标题、标签、说明、标注等）必须使用**中文**，仅行业通用缩写和专业术语（如 AI、API、SaaS、HTML、CSS、JavaScript、Python 等）可保留英文原文。中英文混排时，英文术语前后加空格，保持排版整洁。

---

## 一、核心风格要素

### 1. 透视与构图
- **等距透视（Isometric Projection）**：采用 30° 等距视角，营造伪 3D 效果，兼具技术感与亲和力
- **水平叙事流**：从左到右的阅读顺序，适合表达时间线、流程、演进等概念
- **中心焦点构图**：核心元素居中放置，周围辅以支撑元素，形成清晰的视觉层级
- **大量留白**：背景使用纯白或极浅灰色，确保主体元素突出、呼吸感充足

### 2. 色彩体系

| 角色 | 色彩 | 用途 |
|------|------|------|
| **主色调** | 科技蓝 (#3B82F6 / #4A90E2) | 信任、智能、稳定，占画面 40-60% |
| **辅助色** | 珊瑚红 (#E74C3C / #EF4444)、翡翠绿 (#27AE60 / #10B981)、琥珀黄 (#F5A623) | 功能区分、状态指示、视觉节奏 |
| **中性色** | 冷灰色系 (#6B7280, #9CA3AF, #E5E7EB) | 背景、结构、次要元素 |
| **点缀色** | 青色荧光 (#00D4FF) | 发光效果、科技感强调 |

**色彩原则**：
- 饱和度适中，偏向柔和的企业级调色板（非纯色，略带灰调）
- 色彩具有功能性：不同颜色代表不同概念/阶段/类别
- 色彩渐变从左到右递增（灰→中性→饱和），强化叙事推进感

### 3. 插画技法
- **扁平化设计 + 微立体感**：主体使用纯色填充，配合柔和阴影营造轻微立体感
- **几何简化**：人物和物体被简化为基本几何形体（圆柱形肢体、球形关节、块状躯干）
- **无面部特征人物**：避免人口统计学特征，最大化受众认同感
- **图标化表达**：概念用简洁的图标/符号表示（如盾牌=安全、植物=成长、齿轮=工程）
- **发光效果**：关键元素添加柔和的荧光/辉光，暗示"活跃"、"智能"、"能量"
- **一致的线条粗细**：使用细而均匀的描边，保持画面整洁

### 4. 视觉隐喻库

| 隐喻类型 | 视觉元素 | 表达概念 |
|----------|----------|----------|
| **交通/道路** | 道路、马匹、车辆 | 流程、演进、基础设施、能力 |
| **围栏/护栏** | 护栏柱、围栏带、盾牌 | 治理、约束、安全边界、保护 |
| **平台/基座** | 多层平台、电路板基座 | 技术栈、抽象层级、基础设施 |
| **自然元素** | 植物、生长、循环箭头 | 可持续性、成长、迭代、循环 |
| **数字元素** | 二进制代码、芯片、数据流 | 数字化、数据处理、技术内核 |
| **人体/大脑** | 大脑、神经网络 | 智能、决策、AI 核心 |

### 5. 字体排版
- **中文标题字体**：思源黑体（Noto Sans SC）/ 微软雅黑，粗体
- **英文/术语字体**：Inter、Roboto、SF Pro 等无衬线字体
- **层级分明**：通过字重和颜色对比建立信息层级（标题深色粗体 → 副标题灰色常规 → 标签浅灰）
- **充足的字间距**：保持现代感和可读性
- **避免全大写**：使用首字母大写格式（Sentence case），保持亲和力
- **中英文混排**：英文术语前后加空格（如"AI 治理"、"API 接口"），保持视觉节奏

## 二、构图模式

### 模式 A：中心焦点构图
- **适用**：封面图、核心概念展示、单一主题强调
- **特征**：核心元素居中，周围环绕支撑元素，放射状或对称布局
- **示例**：中心发光大脑 + 六角围栏

### 模式 B：水平叙事流
- **适用**：时间线、流程图、演进对比、阶段展示
- **特征**：从左到右排列，色彩饱和度递增，底部有连接路径
- **示例**：三阶段演进（灰马 → 终端 → 蓝马）

### 模式 C：多元素并列
- **适用**：多概念对比、分类展示、框架概述
- **特征**：等距排列的多个元素，各自独立但共享基座/背景
- **示例**：四柱并列（蓝/红/黄/绿四色柱体）

### 模式 D：层叠递进
- **适用**：技术栈展示、架构层次、流程步骤
- **特征**：从下到上或从后到前的层叠结构，每层代表不同抽象级别
- **示例**：多层平台 + 中心核心元素

## 三、质量检查清单

生成配图后，使用以下清单验证风格一致性：

- [ ] **语言**：图片中所有文字是否为中文？（仅专业术语可保留英文）
- [ ] **透视**：是否使用等距透视（30° 视角）？
- [ ] **背景**：是否为纯白或极浅灰？是否有充足留白？
- [ ] **色彩**：主色是否为科技蓝？辅助色是否为企业级柔和色调？
- [ ] **人物**：人物是否为无面部特征的几何简化风格？
- [ ] **阴影**：阴影是否柔和、一致？
- [ ] **图标**：概念是否用简洁的图标/符号表达？
- [ ] **文字区域**：是否预留了文字标注区域？文字是否清晰可读？
- [ ] **比例**：是否符合目标用途的比例要求？
- [ ] **一致性**：同一组配图的风格是否统一？
- [ ] **中英文混排**：英文术语前后是否有空格？排版是否整洁？
